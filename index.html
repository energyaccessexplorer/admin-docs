    The above diagrams shows the models used by the platform: a geography combined with a category spawns a dataset.

    A dataset itself has a collection of files.

    Each of these entities has several attributes and configuratitons depending on their nature. This document explains the non-trivial aspects of these entities. The basic definitions of the attributes are usually found on the CMS with a hoverable question mark (?) that, when hovered, show the description.

    Geography

    They represent a region on the world. Generally a country (adm0), state/province (adm1) or municipality/county (adm2).

    Each may have a "parent" Geography which is another Geography on the database. Naturally, an adm1 would have and adm0 as a parent but not the other way around.

    All geographies within a country should have the same CCA3 attribute to the country containing them.

    Configuration of a geography is a JSON document which visual and/or functional aspects on the tool. Consider the following JSON object: 

    {
  "boundaries_name": "Subcounty",

  "timeline": true,

  "timeline_dates": [
    "2020-10-31",
    "2021-10-31",
    "2022-10-31"
  ],

  "flag": {
    "x": -4,
    "y": 0,
    "width": 24,
    "height": 16,
    "aspect-ratio": "none"
  },

  "sort_branches": [
    "census",
    "demand",
    "supply",
    "other"
  ],

  "sort_subbranches": [
    "demographics",
    "productive-uses"
  ],

  "sort_datasets": [
    "mines",
    "healthcare",
    "poverty",
    "population"
  ]
}

    boundaries_name - The name to use for the administrative boundary (eg. Province or State)

    timeline sets the variant of the tool to timeline (eg. India) if true. Else, the default variant (eg. Africa) is used.

    timeline_dates a list of dates (formatted as YYYY-MM-DD) for the tool to consider when data from the timeline-indicator datasets CSV's files. This requires timeline to be true.

    flag given the proportions and aethetics of flags of different countries, each requires to be scaled (width and height) or centered (x and y) differently. Lastly aspect-ratio can also be configured and defaults to none (explained here), These values are empirical and depend on taste. See different country geographies for examples on the CMS and the results on the select screen. This only applies for geographies adm0.

    sort_branches sets how the specified branches are shown first on the tool's control panel. The rest of the branches' order is not guaranteed.

    sort_subbranches sets how the specified subbranches are shown first on the tool's control panel. The rest of the subbranches' order is not guaranteed.

    sort_datasets sets how the specified categories (by name) are shown first on the tool's control panel. The rest of the datasets' order is not guaranteed.

    Vectors

    shape_type specifies one of points, polygons or lines.

    fill the inner color of the shape. It does not apply to lines.

    opacity the transparency factor of the shape (0 is fully transparent and 1 is fully opaque).

    stroke the color of the shape's outline.

    stroke-width the width of the shapes stroke. It does not apply to points.

    width/size is different depending on the shape: The radius of the circles representing points It does not apply to lines. It does not apply to polygons.

    dasharray It does not apply to points.

    Analysis

    An analysis is a heatmap or scalar field compromising one or more datasets' rasters. These dataset rasters MUST HAVE the same dimensions (width × height). An example is the easiest way to explain.

    index the part of the analysis where a category contributes to. It can be one of eai, ani, demand, supply, null

    weight a number between 1 and 5 (defaulting to 2) setting the weight/importance of the category.

    clamp given the current domain, how to manage the values greater or lesser than the domain. If not, they are clipped. See this for a formal description.

    invervals WIP.

    indexes each category can behave differently given an analysis. This is an array of object describing the different scenarios. Consider the following JSON: 

    [
  {
    "index": "demand",
    "invert": true,
    "functionality": "linear"
  },

  {
    "index": "eai",
    "invert": false,
    "functionality": "exclusion-buffer"
  }
]
    This would imply that the raster would:
    1. Have no impact in a ani or supply analysis.
    2. Invert it's range in the analysis whilst keeping a linear scale in eai.
    3. Exclude the selected domain whilst on supply.

    Dataset

    A dataset is, initially, the combination between a dataset and a geography - plus some specific attributes and a collection of files.

    category_id The UUID of the category describing the dataset.

    geography_id The UUID of the country that the dataset belongs to.

    name A "programmer-friendly" string that describes to differentiate from other datasets that have the same category.

    name_long A "human-friendly" string which will appear to the users of the tool.

    online If false, the dataset will not be published on the tool.

    label a string to facilitate searching.

    endpoint the complete URL where the file is located.

    Configuration of a file is a JSON document optionally containing:

    • key - an identifier in the GEOJSON that references to a unique value in a column of the CSV (see dataset->configuration->column)

    Datasets + Files

    A single file can be assigned to several datasets. For example, timeline-indicators and boundaries datasets use the same polygons but different CSVs files. This association is done by category_id and file_id.

    category_id the UUID of the category.

    file_id the UUID of the file.

    active only one file of a specific func can be active at any given time. The rest are ignored

    func one of null, raster, vectors, csv
